**PROJECT TYPE:**
Design a high-fidelity web application prototype.

**PROJECT TITLE:**
Arts Policy Lab: The Cultural Budget Year

**PROJECT SUMMARY:**
Create a deeply immersive, simulation-based educational website that places students in the role of an arts policy decision-maker. The interface should feel like a professional policy dashboard combined with a strategy game. The system operates as a month-by-month simulation where user decisions affect a dynamic cultural ecosystem. The experience should communicate complexity, tradeoffs, and feedback loops in arts policy rather than presenting a linear narrative.

---

## OVERALL DESIGN GOALS

Design an interface that feels:
• Strategic and analytical, like a policy command center
• Visually engaging but grounded in institutional aesthetics
• Clear, legible, and information-dense without feeling overwhelming
• Interactive and reactive — the system should feel alive

The product should look like a hybrid between:
• A city simulation dashboard
• A data visualization platform
• A civic technology tool

Avoid gamified clichés (no cartoonish visuals). The tone should feel credible, educational, and contemporary.

---

## PRIMARY USER

College students studying arts, cultural policy, public policy, or media studies.

## SECONDARY USER

Instructors using the simulation in a classroom environment for group competition or discussion.

---

# CORE EXPERIENCE STRUCTURE

Design a full end-to-end flow including the following screens:

1️⃣ Landing / Introduction
2️⃣ Simulation Dashboard (main interface)
3️⃣ Monthly Decision Modal
4️⃣ Stakeholder Inbox
5️⃣ Cultural Ecosystem Visualization
6️⃣ Metrics & Outcomes Panel
7️⃣ End-of-Year Results / Comparison Screen

---

# 1. LANDING SCREEN

PURPOSE
Introduce the simulation and establish tone.

LAYOUT
• Large headline introducing the role:
“You are the Cultural Policy Director.”
• Short paragraph explaining the simulation concept
• Start Simulation button
• Optional: Join Class / Enter Team Code
• Background visual: abstract map or cultural network visualization

STYLE
Editorial, minimal, confident typography
Subtle motion or gradient background

---

# 2. MAIN SIMULATION DASHBOARD (MOST IMPORTANT SCREEN)

This is the central “Policy Desk.”

LAYOUT STRUCTURE
A three-column grid:

LEFT COLUMN — MONTH & SIGNALS
• Current month indicator
• Headline news cards
• Budget summary
• Quick status alerts

CENTER — CULTURAL ECOSYSTEM VIEW
• Interactive map or network diagram
• Visual indicators showing venues, communities, institutions
• Animated changes when decisions are made

RIGHT COLUMN — POLICY CONTROLS
• Funding allocation sliders
• Policy toggles
• Decision buttons
• “Finalize Month” CTA

The dashboard should feel like an operational control center.

---

# 3. MONTHLY DECISION MODAL

When the user enters a new month, a modal appears presenting:

• Context summary (economic climate, political environment)
• Key decision prompt
• Multiple policy options
• Expected tradeoffs visualization

The modal should feel like a briefing document.

---

# 4. STAKEHOLDER INBOX

A messaging panel with memos from different groups:

• Museum director
• Artist collective
• Community coalition
• City council
• Donor foundation

Each message includes:
• Tone indicator (supportive, neutral, critical)
• Short message preview
• Expanded view when clicked

The tone should shift visually (color accents or sentiment markers).

---

# 5. CULTURAL ECOSYSTEM VISUALIZATION

This is a dynamic visual representation of the arts environment.

Options:
• Stylized city map
OR
• Network graph of cultural actors

Visual changes should include:
• Growth or decline indicators
• Activity density
• Geographic distribution of resources

This visualization is the primary feedback channel.

---

# 6. METRICS PANEL

Display key simulation indicators:

• Public Trust
• Artist Livelihood
• Access & Equity
• Cultural Vitality
• Financial Sustainability
• Political Capital

Use:
• Clean charts or gauges
• Color-coded states
• Trend arrows

Metrics should update visibly after each decision.

---

# 7. END-OF-YEAR RESULTS SCREEN

A reflective summary of outcomes.

Include:
• Final metric breakdown
• Narrative summary of leadership style
• Achievements or consequences
• Comparison to class averages (optional)
• Restart simulation button

Tone should feel analytical and reflective rather than celebratory.

---

# INTERACTION DESIGN PRINCIPLES

The system should emphasize feedback loops:

• Decisions immediately change metrics
• Visual ecosystem reacts
• Stakeholder sentiment updates
• New constraints appear

The interface should communicate cause and effect clearly.

---

# VISUAL STYLE

COLOR PALETTE
Muted, institutional tones with one accent color
Examples: deep navy, slate, off-white, muted gold or teal

TYPOGRAPHY
Modern editorial sans serif for UI
Serif or distinctive display font for headlines

ICONOGRAPHY
Minimal, line-based icons

VISUAL MOOD
Serious, thoughtful, contemporary civic tech
